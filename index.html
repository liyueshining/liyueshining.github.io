SpringBoot Filter and Interceptor

Filter and Interceptor Intruduction

Filter and Interceptor 都是aop的实现方式，两者的区别如下：
1、Filter是依赖于Servlet容器，属于Servlet规范的一部分，而拦截器则是独立存在的，可以在任何情况下使用。
2、Filter的执行由Servlet容器回调完成，而拦截器通常通过动态代理的方式来执行。
3、Filter的生命周期由Servlet容器管理，而拦截器则可以通过IoC容器来管理。

Filter的执行顺序

通过注解的方式实现的Filter并没有指定执行的顺序，但是却在通过注册的方式实现的Filter之前执行。@WebFilter这个注解并没有指定执行顺序的属性，其执行顺序依赖于Filter的名称，是根据Filter类名（注意不是配置的filter的名字）的字母顺序倒序排列，并且@WebFilter指定的过滤器优先级都高于FilterRegistrationBean配置的过滤器。

@Scheduled 定时任务的执行方式

fixedDelay

指定两次任务执行的时间间隔(毫秒)，此时间间隔指的是，前一次任务结束与下一个任务开始的间隔。如：@Scheduled(fixedDelay = 5*1000 )，表示第一个任务结束后，过5秒后，开始第二个任务。

fixedRate

指定两次任务执行的时间间隔(毫秒)，此时间间隔指的是，前一个任务开始与下一个任务开始的间隔。如：@Scheduled(fixedRate= 5*1000 )，表示第一个任务开始后(第一个任务执行时间小于5秒)，第一个任务开始后的第6秒，开始第二个任务。如果第一个任务执行时间大于5秒，第一个任务结束后，直接开始第二个任务。

cron

使用表达是进行任务的执行，例如：@Scheduled(cron = “0/15 ? “)每隔15秒执行一次.

初始Jenkins pipeline

2018年圣诞节入职了新单位，刚上来第一件事就是帮团队上线一个新项目，要求是尽量要自动化，方便以后做推广。
毫无疑问，Jenkins是做CI/CD的最好的工具了，之前在ZTE工作时，也在开发之余负责搭建了团队自己的CI流水线，用的就是Jenkins，不过当时pipeline还没有出来。这次主要是基于Jenkins pipeline来做。
做之前大概了解了一下pipeline的语法，主要是脚本式和声明式，脚本式主要是通过自己编写groovy代码来实现pipeline的功能，而声明式pipeline主要是使用现成的groovy编写的好的DSL语句来实现功能。
具体的语法参见： pipeline syntax

Jenkins中如何创建pipeline

pipeline是Jenkins 2.x版本新增的功能，在新建任务时会有选项：创建一个流水线 或是 Pipeline.
任务创建好了之后就可以在pipeline script 中编写pipeline脚本了，或者选择 Jenkins file，从git仓库中读取写在Jenkins file 中的pipeline脚本，需要说明的是读取Jenkins file时 是不拉取git工程到workspace的，应该是读到Jenkins的内存中的，定位Jenkins file的层次时是不带根目录的。
这里遇到过一个坑，定位Jenkins file时带上了根目录，结果报 no such file!

Jenkins中报command not found的问题

一般情况下，在Linux系统上 Jenkins都是运行在jenkins用户下的，切Jenkins调用Linux系统的命令的时候是非登录式的，所以在执行命令之前需要先 source ~/.bash_profile文件，变为登录式执行命令。而且要在Jenkins用户的~/.bash_profile中将要执行的命令加到PATH中。

Jenkins重启后某些job不见了

有天临近下班的时候 机房突然断电了，重启后Jenkins上的很多job都不见了，看了网上很多帖子说是安装了一些版本不兼容的插件，会导致重启后job不见的情况，跟我的情况很像，好像前几天安装了一些高版本的插件，Jenkins不重启没有影响，重启后就惨了。解决方案就是，升级Jenkins到最新版本2.150.

Insert Sort

Insert Sort

排序算法：默认采用升序，两个遍历，第一个是全范围的遍历，
第二个是首元素和当前元素之前的小范围遍历，使元素在该小范围内保持有序，
如果第二个遍历中的当前元素大于第一个遍历中的当前元素，
则将该元素及后面的元素整体向后移动一位，并小的元素插入到第二个遍历中当前元素的位置。

时间复杂度：O(N^2)

空间复杂度：O(1)

关于Bcrypt

bcrypt是一种跨平台的文件加密工具。bcrypt 使用的是布鲁斯·施内尔在1993年发布的 Blowfish 加密算法。由它加密的文件可在所有支持的操作系统和处理器上进行转移。它的口令必须是8至56个字符，并将在内部被转化为448位的密钥
